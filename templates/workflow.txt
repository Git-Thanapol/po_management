│ 9. การตรวจสอบเพิ่มเติม (Sanity Checks) – ทำหลังได้ผลลัพธ์            │
│    • สลับป้าย Label ใน train แล้วฝึกใหม่ → accuracy ควร ≈ random     │
│    • ใช้โมเดลอย่างง่าย (spectral features + Logistic Regression)     │
│      เป็น baseline เปรียบเทียบ                                      │
│    • t‑SNE / PCA ของ embedding สีตาม class และสีตาม session         │
│      เพื่อดูการ clustering ที่ผิดปกติ                                │
│    • ถ้ามีหลายเซสชัน: ทดสอบ Domain Generalization                   │
│        - train บนเซสชัน A, test บนเซสชัน B                         │
│    • Gradient‑CAM: ตรวจสอบว่าโมเดลสนใจความถี่ที่เกี่ยวข้องกับใบพัด   │
